Backend Architecture: DDD Decoupling

Moving from Anemic to Rich Domain Model

Current (Mixed Responsibilities)
  • model/user.go handles hashing via GORM hooks.
  • UserService performs validation logic.
  • Repositories return ORM models directly.
  • Risk: Business logic leaks into Infra.
Target (Strict DDD)
  • entity/user.go owns HashPassword().
  • entity types are returned by Repositories.
  • Application Layer only orchestrates.
  • Benefit: Testable, technology-agnostic logic.
A

Remove GORM Hooks

Clean up model/user.go and shift hashing to UserService via entity.

B

Enforce Entity Mapping

Ensure all DB repositories map 100% to Domain Entities before returning data.
