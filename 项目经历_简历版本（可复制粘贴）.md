## 项目经历

### 高并发社区论坛系统（Bluebell）| 2 个月 | Go、Gin、GORM、MySQL、Redis、Docker

**主导架构设计与底层基建**：基于 Go + Gin + GORM 搭建高并发社区论坛后端微服务架构。采用 DDD 分层设计（Repository → Service → Handler），实现业务逻辑与框架解耦。配置 MySQL 连接池参数（MaxOpenConns: 200, MaxIdleConns: 30），结合多层缓存策略（Redis + 本地缓存），压测环境下单节点成功支撑 **1491 req/s** 的高并发访问，接口 P99 耗时稳定在 **89ms** 以内，缓存命中率达到 **85%+**。

**攻克高并发实时投票与热度排行难题**：摒弃传统关系型数据库的低效聚合操作，基于 Redis ZSet（有序集合）数据结构，设计并实现了帖子的时间线排序与分数（赞/踩）统计算法，支持 Snowflake 算法生成分布式全局唯一 ID。对比传统 SQL 聚合方案，投票查询性能提升 **20-50 倍**，热榜查询接口响应时间从 100-200ms 缩短至 **5-10ms**。

**构建高可靠的安全机制与服务稳定性保障**：独立实现基于 JWT 的身份认证与双 Token（Access/Refresh）无感刷新方案，Access Token 有效期 120 分钟，Refresh Token 有效期 168 小时。在路由层引入基于令牌桶算法的限流中间件（fill_interval: 10ms, capacity: 200），有效防止恶意请求击穿系统。集成 Zap 高性能日志库与自定义 errorx 错误处理栈，实现错误链路精准追踪，线上故障定位效率提升 **80%+**（从 30-60 分钟 → 5-10 分钟）。

**推进工程规范化与云原生容器化部署**：引入 Makefile 与 Air 工具链实现本地环境热重载，集成 Swagger 自动生成 RESTful API 文档，使得前后端联调沟通成本降低 **75%+**。编写 Dockerfile 采用多阶段构建（Builder + Runtime），镜像体积从 400-500 MB 缩小至 **15-20 MB**（缩小 95%）；编写 docker-compose 脚本完成 MySQL、Redis、应用服务的容器化编排，包含完整的健康检查和服务依赖管理，为后续整体迁移至 Kubernetes 集群奠定了标准化基础。
