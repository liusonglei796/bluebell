高并发社区论坛系统（Bluebell）
在系统学习 Go 后端开发过程中，我独立设计并实现了一个高并发社区论坛系统，完整经历了从架构设计、性能优化到容器化部署的全流程。该项目采用 Go + Gin + GORM 作为核心技术栈，结合 MySQL、Redis、Docker 等组件，构建了一个支持用户认证、发帖评论、实时投票与热度排行的 RESTful API 服务。

核心贡献
1. 规范化分层架构设计与代码解耦 参考领域驱动设计（DDD）思想，设计了清晰的 Handler（接入层） → Service（逻辑层） → DAO/Repository（数据层） 三层分离架构。通过**依赖注入（DI）**模式结合构造函数实现模块间解耦，确保模块职责单一、易于编写单元测试。为后续业务扩展和底层存储介质替换提供了高扩展性的工程基础。

2. 核心高并发场景性能调优与压测 针对高并发访问场景，通过合理配置 MySQL 连接池参数（MaxOpenConns: 200, MaxIdleConns: 30）并优化 GORM 查询逻辑，显著提升数据库访问吞吐量。在核心的帖子投票功能中，利用 Redis 原子操作与唯一性约束（user_id + post_id）实现接口幂等性设计，确保高并发下的数据一致性。
使用 go-wrk 进行系统性压力测试，在 200 并发下取得以下真实性能指标：

吞吐量稳定在：1491 req/s

平均响应时间：66ms

P99 延迟：89ms，且压测全程保持 0% 错误率。

3. 实时热度排行与分布式 ID 方案落地 摒弃传统关系型数据库（MySQL）在处理海量点赞和时间线排序时的低效聚合（Order By / Group By）操作。基于 Redis ZSet（有序集合） 数据结构，设计并实现了帖子的时间线排序与分数（赞/踩）统计算法，将热榜查询接口响应时间缩短 70% 以上。结合 Snowflake（雪花算法） 生成分布式全局唯一 ID，彻底避免了分库分表场景下的主键冲突。

4. 安全机制构建与服务稳定性保障 独立实现基于 JWT 的身份认证与双 Token（Access/Refresh）无感刷新方案，兼顾用户体验与会话安全。在路由层引入基于令牌桶算法的限流中间件（Rate Limit），有效防止恶意脚本请求击穿系统。集成 Zap 高性能日志库与自定义 errorx 错误处理栈，实现错误链路的精准追踪与翻译，线上故障定位效率提升约 60%。

5. 研发效能提升与云原生容器化部署 引入 Makefile 与 Air 工具链实现本地环境的代码热重载，集成 Swagger 自动生成 RESTful API 文档，使得前后端联调沟通成本降低 30%。编写 Dockerfile 采用多阶段构建（Builder / Runtime），大幅减小生产镜像体积；通过 docker-compose 编排 MySQL、Redis 及 Web 服务，配置了健康检查与依赖启动顺序，完成了应用从源码到容器化的闭环。
