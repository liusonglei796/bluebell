# 第03章: 用户注册业务流程设计

> **学习目标**: 掌握 DDD 分层架构(Handler-Service-DAO)的设计思想,理解注册业务的完整流程,学会职责分离与依赖注入的编程范式。

---

## 1. DDD 分层架构设计模式

### 1.1 各层职责详解

#### Handler 层 (请求处理)

```go
// internal/handler/user_handler.go

func (h *Handlers) SignUpHandler(c *gin.Context) {
    p := new(request.SignUpRequest)
    if err := c.ShouldBindJSON(p); err != nil {
        backfront.HandleError(c, errorx.ErrInvalidParam)
        return
    }

    if err := h.Services.User.SignUp(c.Request.Context(), p); err != nil {
        backfront.HandleError(c, err)
        return
    }

    backfront.ResponseSuccess(c, nil)
}
```

#### Service 层 (业务逻辑)

```go
// internal/service/user/user_service.go

func (s *UserService) SignUp(ctx context.Context, p *request.SignUpRequest) error {
    if err := s.userRepo.CheckUserExist(ctx, p.Username); err != nil {
        return err
    }

    userID := snowflake.GenID()

    u := &model.User{
        UserID:   userID,
        UserName: p.Username,
        Passwd:   p.Password, // 密码加密由 Model 的 BeforeCreate 钩子完成
    }

    return s.userRepo.InsertUser(ctx, u)
}
```

#### DAO 层 (数据访问)

```go
// internal/dao/mysql/user.go

func (r *UserRepo) CheckUserExist(ctx context.Context, username string) error {
    var count int64
    err := r.db.WithContext(ctx).Model(&model.User{}).
        Where("user_name = ?", username).Count(&count).Error
    if err != nil {
        return err
    }
    if count > 0 {
        return repository.ErrUserExist
    }
    return nil
}
```

## 2. 依赖注入与装配

### 2.1 main.go 中的装配

```go
// cmd/bluebell/main.go

// 1) 创建 UnitOfWork 实例（DAO 层）
uow := mysql.NewRepositories(gormDB)

// 2) 创建 Cache Repository 实例
voteCache := redis.NewVoteCache()
tokenCache := redis.NewUserTokenCache()

// 3) 创建 Services 聚合器，注入 UnitOfWork 和 Cache
// VoteCache 同时实现了 VoteCacheRepository 和 PostCacheRepository
services := service.NewServices(uow, voteCache, voteCache, tokenCache, cfg)

// 4) 创建 Handlers 聚合器，注入 Services
handlers := handler.NewHandlers(services)

// 5) 注册路由，注入 Handlers
r, err := router.NewRouter(cfg.App.Mode, handlers, cfg)
```
