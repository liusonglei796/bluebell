# 第18章:按社区筛选帖子实现

> **本章导读**
>
> 现在我们已经实现了全站帖子的投票和排序,但用户还需要一个重要功能:**按社区筛选帖子**。
> 本章将学习如何设计统一的帖子查询接口,实现按社区筛选帖子的完整流程。

## 1. 参数设计

我们通过 `request.PostListRequest` 统一获取参数。

#### 文件: `internal/dto/request/post.go`

```go
type PostListRequest struct {
	Page        int64  `json:"page" form:"page"`
	Size        int64  `json:"size" form:"size"`
	Order       string `json:"order" form:"order"`
	CommunityID int64  `json:"community_id" form:"community_id"`
}
```

---

## 2. Logic 层实现 - 调度器模式

Service 层会根据 `CommunityID` 是否为 0 来决定查询逻辑。

#### 文件: `internal/service/post/post_service.go`

```go
// GetCommunityPostList 根据社区ID获取帖子列表
func (s *postServiceStruct) GetCommunityPostList(ctx context.Context, p *request.PostListRequest) (data []*response.PostDetailResponse, err error) {
	// 1. 从 Redis 获取特定社区按顺序排列的帖子 IDs
	// 注意：p.CommunityID 此时传入的是社区的自增 ID
	ids, err := s.postCache.GetCommunityPostIDsInOrder(ctx, p.CommunityID, p.Order, p.Page, p.Size)
	if err != nil {
		zap.L().Error("postCache.GetCommunityPostIDsInOrder failed", zap.Error(err))
		return nil, errorx.ErrServerBusy
	}

	if len(ids) == 0 {
		return make([]*response.PostDetailResponse, 0), nil
	}

	// 2. 批量查 MySQL 和 批量查投票 (复用已有逻辑)
	posts, err := s.postRepo.GetPostListByIDsWithPreload(ctx, ids)
	// ...
}
```

注意：全站查询和社区查询在 Service 层被拆分为两个独立的方法 `GetPostList` 和 `GetCommunityPostList`，由 Handler 层根据 `p.CommunityID` 是否为0进行分发调用。

## 3. DAO 层实现

#### 文件: `internal/dao/mysql/post.go`

由于查询逻辑已经转移到 Redis，MySQL 这里不需要针对社区提供专门的查询方法。

我们直接复用第15章实现的 `GetPostListByIDsWithPreload` 方法即可，因为 Redis 已经帮我们把社区过滤好了！
