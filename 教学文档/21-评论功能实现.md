## 1. 评论业务规则设计

### 1.1 核心规则

**规则1: 评论归属**
- 每条评论必须关联一个帖子
- 每条评论必须有一个作者

**规则2: 内容要求**
- 评论内容不能为空
- 支持文本格式

**规则3: 查看权限**
- 所有登录用户都可以查看评论
- 评论列表按时间倒序排列

### 3.2 MySQL 实现

#### 文件: `internal/dao/database/postdb/remark.go`

```go
package postdb

import (
	"bluebell/internal/model"
	"bluebell/pkg/errorx"
	"context"
	"gorm.io/gorm"
)

// remarkRepoStruct 评论数据访问层实现
type remarkRepoStruct struct {
	db *gorm.DB
}

// NewRemarkRepo 创建评论仓储实例
func NewRemarkRepo(db *gorm.DB) *remarkRepoStruct {
	return &remarkRepoStruct{db: db}
}

// CreateRemark 创建评论
func (r *remarkRepoStruct) CreateRemark(ctx context.Context, remark *model.Remark) error {
	if err := r.db.WithContext(ctx).Create(remark).Error; err != nil {
		return errorx.Wrap(err, errorx.CodeDBError, "create remark failed")
	}
	return nil
}

// GetRemarksByPostID 获取帖子的所有评论
func (r *remarkRepoStruct) GetRemarksByPostID(ctx context.Context, postID int64) ([]*model.Remark, error) {
	var remarks []*model.Remark
	err := r.db.WithContext(ctx).
		Where("post_id = ?", postID).
		Preload("Author").
		Order("created_at DESC").
		Find(&remarks).Error
	if err != nil {
		return nil, errorx.Wrap(err, errorx.CodeDBError, "get remarks by post id failed")
	}
	return remarks, nil
}
```

**实现要点:**
1. **Preload("Author")**: 使用 GORM Preload 预加载评论作者信息,避免 N+1 查询问题
2. **Order("created_at DESC")**: 按创建时间倒序排列,最新评论显示在最前面
3. **errorx.Wrap**: 统一包装数据库错误,携带业务错误码

### 4.2 业务逻辑实现

#### 文件: `internal/service/postsvc/post_service.go`

```go
// RemarkPost 发表评论
func (s *postServiceStruct) RemarkPost(ctx context.Context, req *postreq.RemarkRequest, userID int64) error {
	// 1. 校验帖子是否存在
	post, err := s.postRepo.GetPostByID(ctx, req.PostID)
	if err != nil {
		zap.L().Error("remarkPost: postRepo.GetPostByID failed",
			zap.Int64("post_id", req.PostID),
			zap.Error(err))
		return errorx.ErrServerBusy
	}
	if post == nil || post.PostID == "" {
		return errorx.ErrNotFound
	}

	// 2. 构建评论模型
	remark := &model.Remark{
		PostID:   req.PostID,
		Content:  req.Content,
		AuthorID: userID,
	}

	// 3. 保存到数据库
	if err := s.remarkRepo.CreateRemark(ctx, remark); err != nil {
		zap.L().Error("remarkPost: remarkRepo.CreateRemark failed",
			zap.Int64("post_id", req.PostID),
			zap.Int64("author_id", userID),
			zap.Error(err))
		return errorx.ErrServerBusy
	}

	return nil
}

// GetPostRemarks 获取帖子评论列表
func (s *postServiceStruct) GetPostRemarks(ctx context.Context, postID int64) ([]*postResp.RemarkDetail, error) {
	// 1. 获取原始评论列表
	remarks, err := s.remarkRepo.GetRemarksByPostID(ctx, postID)
	if err != nil {
		zap.L().Error("getPostRemarks: remarkRepo.GetRemarksByPostID failed",
			zap.Int64("post_id", postID),
			zap.Error(err))
		return nil, errorx.ErrServerBusy
	}

	// 2. 转换为 DTO
	resp := make([]*postResp.RemarkDetail, 0, len(remarks))
	for _, r := range remarks {
		authorName := "已注销用户"
		if r.Author != nil {
			authorName = r.Author.UserName
		}
		resp = append(resp, &postResp.RemarkDetail{
			ID:         r.ID,
			Content:    r.Content,
			AuthorName: authorName,
			CreateTime: r.CreatedAt,
		})
	}

	return resp, nil
}
```

**业务逻辑要点:**
1. **帖子存在性校验**: 评论前必须验证帖子是否存在
2. **DTO 转换**: 将数据库模型转换为响应 DTO,隐藏敏感信息
3. **作者名称处理**: 如果作者不存在,显示"已注销用户"
4. **错误日志**: 记录详细的错误日志,便于排查问题

## 5. Handler 层实现

### 5.1 发表评论接口

```go
// PostRemarkHandler 处理发表评论请求
func (h *Handler) PostRemarkHandler(c *gin.Context) {
	// 1. 获取当前用户 ID
	userID, exist := c.Get("UserIDKey")
	if !exist {
		backfront.HandleError(c, errorx.ErrNeedLogin)
		return
	}

	// 2. 绑定参数
	req := &postreq.RemarkRequest{}
	if err := c.ShouldBindJSON(req); err != nil {
		var errs validator.ValidationErrors
		if errors.As(err, &errs) {
			translatedErrs := errs.Translate(translate.Trans)
			backfront.HandleErrorWithMsg(c, errorx.CodeInvalidParam,
				translate.RemoveTopStruct(translatedErrs))
			return
		}
		backfront.HandleError(c, errorx.ErrInvalidParam)
		return
	}

	// 3. 执行业务
	if err := h.postService.RemarkPost(c.Request.Context(), req, userID.(int64)); err != nil {
		backfront.HandleError(c, err)
		return
	}

	// 4. 响应
	backfront.ResponseSuccess(c, nil)
}
```

## 9. 设计思想总结

### 9.1 数据模型设计

**评论与帖子的关系：**
- 一对多关系：一个帖子可以有多个评论
- 通过 `post_id` 外键关联
- 使用 GORM 的 `foreignKey` 和 `references` 建立关联

**评论与用户的关系：**
- 一对多关系：一个用户可以发表多个评论
- 通过 `author_id` 外键关联
- 使用 GORM Preload 预加载作者信息

---

### 9.2 查询优化

**Preload 预加载：**
```go
// ❌ 错误做法：循环查询（N+1 问题）
for _, remark := range remarks {
    author := getUserByID(remark.AuthorID) // 每次循环都查询数据库
}

// ✅ 正确做法：Preload 预加载（一次查询）
r.db.Preload("Author").Find(&remarks) // 一次查询加载所有关联数据
```

**排序策略：**
- 按 `created_at DESC` 排序
- 最新评论显示在最前面
- 符合用户阅读习惯

---

### 9.3 错误处理

**分层错误处理：**
- **DAO 层**: 使用 `errorx.Wrap` 包装数据库错误
- **Service 层**: 业务逻辑校验，返回具体业务错误
- **Handler 层**: 统一错误响应格式

**错误码设计：**
```go
// 数据库错误
errorx.CodeDBError

// 参数错误
errorx.CodeInvalidParam

// 未找到
errorx.ErrNotFound

// 服务器繁忙
errorx.ErrServerBusy
```

## 11. 知识回顾
