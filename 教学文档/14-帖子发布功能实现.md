## 1. 数据库模型设计

### 1.1 internal/model/post.go

```go
package model

import "gorm.io/gorm"

type Post struct {
	gorm.Model
	PostID      string     `gorm:"column:post_id;not null;primaryKey;size:255"`
	AuthorID    int64      `gorm:"column:author_id"`
	CommunityID int64      `gorm:"column:community_id"`
	PostTitle   string     `gorm:"column:post_title;not null;type:text"`
	Author      *User      `gorm:"foreignKey:AuthorID;references:UserID"`
	Community   *Community `gorm:"foreignKey:CommunityID;references:ID"` // 更改点：引用 Community 的 ID 主键
	Content     string     `gorm:"column:content;type:text;not null"`
	Status      int8       `gorm:"column:status"`
}

func (Post) TableName() string {
	return "post"
}
```

## 3. Service 层实现

#### 文件: `internal/service/post/post_service.go`

```go
func (s *postServiceStruct) CreatePost(ctx context.Context, p *request.CreatePostRequest, authorID int64) (postID int64, err error) {
    // 1. 生成帖子ID (int64)
    postID = snowflake.GenID()

    // 2. 构造 Post 对象
    post := &model.Post{
        PostID:      strconv.FormatInt(postID, 10), // Model 中存储为 string 避免精度问题
        AuthorID:    authorID,
        CommunityID: p.CommunityID,
        PostTitle:   p.Title,
        Content:     p.Content,
        Status:      1,
    }

    // 3. 保存到数据库
    if err = s.postRepo.CreatePost(ctx, post); err != nil {
        // DAO 层返回的错误已被 errorx.Wrap 包装，直接返回
        return 0, err
    }
    return postID, nil
}
```

> **错误处理说明**: Service 层直接返回 DAO 层返回的错误。DAO 层使用 `errorx.Wrap()` 包装数据库错误。
