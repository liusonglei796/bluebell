# 第15章:帖子列表分页实现

> **本章导读**
>
> 帖子可以创建和查看详情了,但用户还需要一个**帖子列表页**来浏览所有帖子。
> **分页(Pagination)**是解决大数据量展示的经典方案。

---

## 1. 分页请求参数

我们通过 `request.PostListRequest` 接收分页和排序参数。

#### 文件: `internal/dto/request/post.go`

```go
type PostListRequest struct {
	Page        int64  `json:"page" form:"page"`
	Size        int64  `json:"size" form:"size"`
	Order       string `json:"order" form:"order"`

}
```

## 2. Service 层实现

Service 层负责协调数据查询和多表关联。

#### 文件: `internal/service/post/post_service.go`

```go
func (s *PostService) GetPostList(ctx context.Context, p *request.PostListRequest) (data []*response.PostDetailResponse, err error) {
	// 1. 获取帖子列表
	posts, err := s.postRepo.GetPostList(ctx, p.Page, p.Size)
	if err != nil {
		return nil, errorx.ErrServerBusy
	}

	// 2. 批量获取投票数据
	postIDs := make([]string, 0, len(posts))
	for _, post := range posts {
		postIDs = append(postIDs, post.PostID)
	}
	voteData, err := s.voteCache.GetPostVoteData(ctx, postIDs)

	// 3. 拼装响应数据 (解决 N+1 问题)
	data = make([]*response.PostDetailResponse, 0, len(posts))
	for idx, post := range posts {
		authorName := "未知用户"
		if post.Author != nil {
			authorName = post.Author.UserName
		}

		postDetail := &response.PostDetailResponse{
			ID:          post.PostID,
			AuthorID:    strconv.FormatInt(post.AuthorID, 10),
			CommunityID: post.CommunityID,
			Status:      post.Status,
			Title:       post.PostTitle,
			Content:     post.Content,
			CreateTime:  post.CreatedAt,
			AuthorName:  authorName,
			VoteNum:     voteData[idx],
		}
		data = append(data, postDetail)
	}
	return data, nil
}
```

## 3. DAO 层实现

#### 文件: `internal/dao/mysql/post.go`

```go
func (r *PostRepo) GetPostList(ctx context.Context, page, size int64) (posts []*model.Post, err error) {
	err = r.db.WithContext(ctx).
		Preload("Author").    // 预加载作者，避免 N+1
		Preload("Community"). // 预加载社区
		Order("created_at DESC").
		Limit(int(size)).
		Offset(int((page - 1) * size)).
		Find(&posts).Error
	return
}
```
