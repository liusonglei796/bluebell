## 1. 密码安全:Web 开发的生命线

### 1.1 永远不要存储明文密码

**这是铁律!** 如果数据库中存储的是明文密码(例如 `123456`),一旦发生数据泄露,所有用户账户将瞬间遭受灭顶之灾。

**✅ 正确的做法:**

```sql
-- 存储加密后的密码 (bcrypt 哈希值)
INSERT INTO user (user_name, passwd)
VALUES ('zhangsan', '$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy');
```

## 2. 密码哈希算法对比

### 2.1 为什么推荐 bcrypt?

**优势: 自动加盐(Salt)**

```go
bcrypt.GenerateFromPassword([]byte("123456"), bcrypt.DefaultCost)
// 第一次: $2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy
// 第二次: $2a$10$K3eP9jLMNO2pQrS5tUvW7.8fH6gT4yX9vZaB1cD2eF3gH4iJ5kL6m
//         ^^^^^^^^^^^^^^^^^^^^^^^^^^ 这部分是随机盐值
```

**每次加密都会生成不同的盐值**,即使两个用户的密码都是 `123456`,哈希值也完全不同,彩虹表失效!

## 3. Model 层实现:GORM BeforeCreate 钩子

### 3.1 密码加密放在 Model 层

在 Bluebell 项目中,我们选择在 **Model 层** 通过 GORM 的 `BeforeCreate` 钩子自动处理密码加密。

```go
// internal/model/user.go

package model

import (
	"golang.org/x/crypto/bcrypt"
	"gorm.io/gorm"
)

const BcryptCost = 10

// User 用户模型
type User struct {
	gorm.Model
	UserID   int64  `gorm:"column:user_id"`
	UserName string `gorm:"column:user_name;size:64;not null"`
	Passwd   string `gorm:"column:passwd;size:255;not null"`
}

// TableName 自定义表名
func (User) TableName() string {
	return "user"
}

// BeforeCreate GORM 钩子，创建前自动处理
func (u *User) BeforeCreate(tx *gorm.DB) error {
	if u.Passwd != "" {
		hash, err := bcrypt.GenerateFromPassword([]byte(u.Passwd), BcryptCost)
		if err != nil {
			return err
		}
		u.Passwd = string(hash)
	}
	return nil
}
```

## 4. DAO 层实现:数据访问

### 4.1 插入用户数据

```go
// internal/dao/mysql/user.go

func (r *userRepoStruct) InsertUser(ctx context.Context, user *model.User) (err error) {
    // GORM Create 会在执行前自动调用 BeforeCreate 钩子
    // → BeforeCreate 中会自动加密密码
    err = r.db.WithContext(ctx).Create(user).Error
    if err != nil {
        // 使用 errorx.Wrap 包装数据库错误，保留错误链
        return errorx.Wrap(err, errorx.CodeDBError, "插入用户失败")
    }
    return nil
}
```

> **最佳实践**: DAO 层使用 `errorx.Wrap()` 包装底层错误，添加业务错误码。

## 5. Service 层实现:业务流程编排

### 5.1 注册业务流程

```go
// internal/service/user/user_service.go

func (s *userServiceStruct) SignUp(ctx context.Context, p *request.SignUpRequest) (err error) {
    // 1. 判断用户是否存在，使用 errorx.GetCode 判断错误码
    if err = s.userRepo.CheckUserExist(ctx, p.Username); err != nil {
        if errorx.GetCode(err) == errorx.CodeUserExist {
            return err
        }
        // 其他数据库错误返回服务器繁忙
        return errorx.ErrServerBusy
    }

    // 2. 生成 UserID (雪花算法)
    userID := snowflake.GenID()

    // 3. 构造 User 实例
    u := &model.User{
        UserID:   userID,
        UserName: p.Username,
        Passwd:   p.Password, // 传入明文，BeforeCreate 会处理加密
    }

    // 4. 保存到数据库
    return s.userRepo.InsertUser(ctx, u)
}
```

> **注意**: Service 层使用 `errorx.GetCode(err)` 判断错误码，而不是 `errors.Is()`。
